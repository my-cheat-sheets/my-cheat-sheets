üìä XDM System (Experience Data Model)
  • Core Concepts 
    Schema: Structure of data (People, Assets)
Class: Behavioral vs Record data (XDM Individual Profile, XDM ExperienceEvent)
Field Groups: Reusable clusters of fields (Demographics, Web Details)
Data Types: Reusable complex types (Person Name, Address)
  • Best Practices 
    - Reuse standard Field Groups before creating custom ones.
- Use 'IdentityMap' for multi-identity data.
- Enable 'Profile' toggle ONLY for schemas needed in Real-Time CDP.
üì• Data Ingestion
  • Methods 
    Batch Ingestion: Uploading CSV/Parquet files via UI or API.
Streaming Ingestion: Real-time data via HTTP API endpoint.
Source Connectors: Pre-built integration (S3, Azure Blob, Google Analytics).
  • Datasets 
    - Container for ingested data.
- Mapped to a specific XDM Schema.
- Enable 'Profile' to hydrate Unified Profile.
üîç Query Service (SQL)
  • Common Commands 
    -- Show available tables (datasets)
SHOW TABLES;

-- Inspect schema
DESCRIBE table_name;

-- Flatten nested XDM fields
SELECT education.school_name 
FROM table_name;

-- Work with Maps (IdentityMap)
SELECT explode(identityMap['email']) 
FROM table_name;

-- CTAS (Create Table As Select) - Output to Dataset
CREATE TABLE new_dataset 
WITH (FORMAT='PARQUET') 
AS SELECT * FROM source_dataset;
  • PSQL Command Line 
    \d  # List tables
\dt # List tables (standard)
\du # List users/roles
üéØ Real-Time CDP
  • Segmentation 
    Segment Definitions: Rules to categorize users (e.g., "Visited > 3 times").
Evaluation:
- Edge: Evaluated instantly on data collection.
- Streaming: Evaluated as data flows in.
- Batch: Evaluated on scheduled interval (daily).
  • Destinations 
    Activates segments to external platforms (Facebook, Google Ads, Email).
Types:
- Dataset Export: S3, Blob storage.
- Streaming: HTTP API forwarding.
üöÄ Adobe Journey Optimizer (AJO)
  • Core Components 
    Journeys: Unitary interactions, real-time logic (e.g., Abandoned Cart).
Campaigns: Scheduled, batch messaging (e.g., Weekly Newsletter).
Offers: Personalized content via Offer Decisioning Engine.
  • Channels 
    - Email (Design via Email Designer)
- Push Notifications (Mobile App)
- SMS
- In-App Messages
- Direct Mail
üìà Customer Journey Analytics (CJA)
  • Architecture 
    Data Views: Virtual reporting suites on top of AEP Datasets.
Connections: Mapping AEP datasets to CJA.
Analysis Workspace: Drag-and-drop analysis UI (same as Adobe Analytics).
  • Key Features 
    - Cross-channel analysis (Online + Offline data).
- Stitching: Merging visitor IDs across devices.
- Derived Fields: Create fields on the fly without schema changes.
üéØ Adobe Target
  • Activity Types 
    A/B Test: Compare two or more versions to see which performs better.
XT (Experience Targeting): Deliver specific content to specific audiences.
MVT (Multivariate Testing): Test combinations of elements.
AP (Automated Personalization): ML-driven personalization.
  • Implementation 
    VEC (Visual Experience Composer): WYSIWYG editor for DOM manipulation.
Form-Based: For non-visual changes (JSON, redirects).
at.js / aloe.js: Client-side libraries.
Server-Side: Delivery API for headless implementation.
üìä Adobe Analytics
  • Variables 
    eVars (Conversion Settings): Persistent variables (e.g., Campaign ID). Expires on event/visit.
Props (Traffic Variables): Hit-level variables (e.g., Page Name). No persistence.
Events (Success Events): Counters/Metrics (e.g., Orders, Logins).
  • Processing 
    Processing Rules: Transform data server-side before storage.
VISTA Rules: Advanced server-side logic (requires engineering).
Marketing Channels: Rules to categorize incoming traffic (SEO, Paid Search).

üîí Privacy & Governance
  • DULE (Data Usage Labeling & Enforcement) 
    Labels: Tag data fields with sensitivity (e.g., I1 for Identity, C1 for Contractual).
Policies: Rules blocking actions based on labels (e.g., "Don't export I1 data to Facebook").
Marketing Actions: Defined purposes for data use (e.g., "Email Targeting").
